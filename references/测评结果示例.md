# 测评结果格式示例

本文件为测评结果的格式参考，每次生成报告时严格按此格式。

---

## 格式规则

1、顶部放评分汇总表格，方便一眼看全局
2、每个模型按顺序放：评论（截图证据）→ 评分 → 交付效率 → 轨迹圈选
3、打分理由拆成「需求 / 代码 / 轨迹」三块，只写大白话，不出现文件名、行号、代码
4、轨迹圈选才写代码位置：文件名:行号用正文（不加反引号），代码片段用代码块，说明用正文
5、轨迹圈选**必须有**，每个模型至少一条。没缺点就写亮点
6、所有列表用 `1、` `2、` 中文顿号序号
7、评论模块直接用截图文件名里的内容
8、交付效率分模型写，仅当多模型效果相近时填写，大白话说明

## cliff

### 评论（截图证据）

1、首页没问题
2、更换主题有 bug，点击另一个主题，选择主题弹窗会继续弹出一次
3、星火主题的排版不对，没有居中
4、点击分享有问题，预期是分享海报页面，然后可以点击保存到本地的

### 评分

总分 6/10：需求 3/5；代码 2/3；轨迹 1/2。封顶：否。

需求：
1、主题切换弹窗有 bug，点另一个主题后弹窗会再弹一次，体验很割裂
2、星火主题排版没居中，语录文字偏左
3、分享直接下载了，没有预览海报的步骤
4、首页、翻书动画、卡片翻转、洗牌袋都正常

代码：
1、有两个文件写了没被引用，死代码
2、用了小程序的标签没改成 HTML 标准标签
3、canvas.js 两个模板函数大量重复，800行

轨迹：
1、星火居中和分享预览这种基本验收没验过就交了

### 交付效率

效率分 7/10（耗时 12min，工具轮次 45，用户轮次 1）
整体还行，中间在主题切换那块试了两种方案

### 轨迹圈选

1、pwa/src/components/TemplateSelector.js:128-133 — 选主题后弹窗重复弹出

```js
container.querySelectorAll('.ts-item').forEach(item => {
    item.addEventListener('click', () => {
      const template = item.dataset.template;
      if (onChange) onChange(template);
      setTimeout(() => renderTemplateSelector(container, { ...options, currentTemplate: window.globalState.currentTemplate }), 50);
    });
  });
```

说明：选主题后 setTimeout 50ms 重新 render 整个面板，导致弹窗再弹一次

2、pwa/src/pages/result.js:438-469 — 分享直接下载没有预览

```js
canvas.toBlob(async (blob) => {
  if (!blob) { alert('生成图片失败'); return; }
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = `毛选语录_${Date.now()}.png`;
  document.body.appendChild(a);
  a.click();
```

说明：没有预览步骤，直接 toBlob 后触发下载。应该先展示海报让用户确认再保存

## gale

需求：
